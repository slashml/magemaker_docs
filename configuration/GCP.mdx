## Overview
This page walks you through **all** the steps required to use Magemaker with **Google Cloud Vertex AI**:

1. Creating / selecting a GCP project
2. Installing & initializing the **gcloud** CLI
3. Enabling **Vertex AI** and related APIs
4. Setting the required **environment variables** (`PROJECT_ID`, `GCLOUD_REGION`, …)
5. (Optional) Creating a dedicated **service-account** for CI/CD deployments

<Note>
Magemaker supports Python&nbsp;≥&nbsp;3.11 (3.12 is currently not supported) and uses the official Vertex AI SDK under the hood.
</Note>

---

<Steps>
  <Step title="Create Google Cloud Account">
    Visit the <a href="https://cloud.google.com" target="_blank">Google Cloud Console</a> and sign up / sign in.
  </Step>

  <Step title="Create (or Select) a Project">
    If this is your first time, the default project is <strong>“My First Project”</strong>. Use the project-picker at the top of the console to create a new project if you prefer a clean slate.
    
    ![Create GCP Project](../Images/google_new_project.png)
  </Step>

  <Step title="Install Google Cloud SDK">
    1. Follow the official <a href="https://cloud.google.com/sdk/docs/install-sdk" target="_blank">installation guide</a> for your OS.<br />
    2. Initialize the SDK:
    ```bash
    gcloud init
    ```
  </Step>

  <Step title="Authenticate & Set Default Project">
    During <code>gcloud init</code> you will:
    - Log in with your browser
    - Pick the project you created earlier

    Verify the default credentials work:
    ```bash
    gcloud auth application-default login
    ```
  </Step>

  <Step title="Enable Required APIs">
    Open <em>APIs & Services → Library</em> and enable **Vertex AI API** for your project.
    
    ![Enable Vertex AI](../Images/QrB_Image_11.png)

    <Note>
      For training or custom container deployments you may also need <code>Cloud Storage</code>, <code>Artifact Registry</code>, and <code>Cloud Build</code>.
    </Note>
  </Step>

  <Step title="(Optional) Create a Service Account">
    For automated pipelines:
    ```bash
    gcloud iam service-accounts create magemaker-sa \
      --description="CI/CD deployments" \
      --display-name="magemaker-sa"

    gcloud projects add-iam-policy-binding $PROJECT_ID \
      --member="serviceAccount:magemaker-sa@$PROJECT_ID.iam.gserviceaccount.com" \
      --role="roles/aiplatform.admin"
    ```
    Download the JSON key and reference it via <code>GOOGLE_APPLICATION_CREDENTIALS</code>.
  </Step>
</Steps>

---

## Environment Variables (.env)
Magemaker loads cloud-specific settings from a **.env** file in your project root. Make sure the following exist for GCP deployments:

```bash
# GCP Configuration
PROJECT_ID="your-project-id"          # gcloud config get-value project
GCLOUD_REGION="us-central1"           # default region for Vertex AI

# (Optional) Path to a service-account JSON key
GOOGLE_APPLICATION_CREDENTIALS="/path/to/magemaker-sa.json"
```

<Warning>
Never commit your <code>.env</code> file to version control!
</Warning>

---

## Verifying Your Setup
```bash
# 1. Configure Magemaker for GCP
magemaker --cloud gcp

# 2. List existing Vertex AI endpoints
magemaker --list
```
If you see an empty list instead of an error, your credentials and environment variables are correct.
