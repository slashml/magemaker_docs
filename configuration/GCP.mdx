# Google Cloud Platform (GCP) Configuration

To use GCP services with Magemaker (e.g., deploying/querying Vertex AI endpoints), you must supply GCP credentials.

## Setting Up GCP Credentials

### 1. Create or Use a Google Cloud Service Account

- Go to **IAM & Admin > Service Accounts**
- Create a service account with Vertex AI and Storage Admin permissions
- Download the JSON key file

### 2. Install Google Cloud CLI

Install via the [official guide](https://cloud.google.com/sdk/docs/install), or let Magemaker's Dockerfile install it for you.

### 3. Provide the Credentials

- **Locally:**
    1. Place your JSON key at `~/.config/gcloud/application_default_credentials.json`
    2. Authenticate:
        ```bash
        gcloud auth application-default login
        # or set env var
        export GOOGLE_APPLICATION_CREDENTIALS="/absolute/path/to/key.json"
        ```
- **Docker:**
    - Mount your gcloud credentials:
      ```bash
      docker run -it \
        -v $HOME/.config/gcloud:/root/.config/gcloud:ro \
        magemaker-server
      ```
    - Docker entrypoint will automatically set `GOOGLE_APPLICATION_CREDENTIALS` if your application default credentials exist.

### 4. Testing Credentials

The `preflight.sh` script provides automated checking:

```bash
bash magemaker/scripts/preflight.sh
```

If everything is setup properly, it will check access to your GCP project.


---

## Environment Variable Summary

- `GOOGLE_APPLICATION_CREDENTIALS` — Path to your Service Account JSON key

---

**Deployment Tip:** After deploying, you'll see a message like this:

```bash
Important: Configuration saved at: /app/configs/<endpoint_name>.yml
You'll need this path for querying the model later.
```

Save or copy this path to use with Magemaker's query command.
