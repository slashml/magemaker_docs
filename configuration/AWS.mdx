## Overview
This guide walks you through the exact AWS prerequisites Magemaker needs **before** you can deploy a model to SageMaker.

<Note>
Running `magemaker --cloud aws` will set **everything** up for you automatically (create an IAM role, write a `.env` file, verify the AWS CLI, etc.).
You only need to perform the manual steps below if you want to understand or replicate what the CLI does under the hood.
</Note>

---
### 1 ‚Äì Install / Verify AWS CLI

‚Ä¢ macOS & Apple Silicon: install via Homebrew and make sure you are **using a Rosetta terminal** (`arch` should print `i386`).  
‚Ä¢ Windows / Linux: follow the official docs.

```bash
# macOS
brew update && brew install awscli
# verify
aws --version
```

Full install instructions:  
https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html

### 2 ‚Äì Create an AWS Account (if you don‚Äôt have one)

1. Go to [aws.amazon.com](https://aws.amazon.com/) and sign up.  
2. Log in to the [AWS Console](https://console.aws.amazon.com/).

---
### 3 ‚Äì Generate an IAM User & Access Keys

<Steps>
<Step title="Navigate to IAM">
Search for **IAM** in the console (‚öÝÔ∏è *do **NOT** select ‚ÄúIAM Identity Center‚Äù*).
</Step>

<Step title="Create or Select User">
1. Click **Users** in the sidebar.  
2. Create a new user (or pick an existing one).
</Step>

<Step title="Attach Policies">
Attach **all three** policies:
- `AmazonSageMakerFullAccess`
- `IAMFullAccess`
- `ServiceQuotasFullAccess`
</Step>

<Step title="Generate Access Keys">
1. In the user details, open **Security credentials**.  
2. Under **Access keys**, click **Create access key** ‚Üí **Command Line Interface**.
</Step>
</Steps>

Store the *Access key ID* **and** *Secret access key* in a safe place‚Äîyou will paste them into the Magemaker prompt shortly.

### 4 ‚Äì (Optional) Understand the Automatic Role Setup

When you run `magemaker --cloud aws` the CLI executes
`magemaker/scripts/setup_role.sh` which:

1. Creates an IAM role called **SageMakerRole** with the following trust policy:
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [{
       "Effect": "Allow",
       "Principal": {
         "Service": "sagemaker.amazonaws.com"
       },
       "Action": "sts:AssumeRole"
     }]
   }
   ```
2. Attaches the managed policy `AmazonSageMakerFullAccess`.
3. Writes the resulting role ARN to `.env` as `SAGEMAKER_ROLE`.

If you prefer to run the script yourself:

```bash
bash magemaker/scripts/setup_role.sh
```

<Warning>
The script requires a configured AWS CLI profile with **admin-level** permissions, otherwise role creation will fail.
</Warning>

---
### 5 ‚Äì Configure Magemaker

Run the interactive setup:

```bash
magemaker --cloud aws
```

You will be prompted for:
1. **AWS Access key ID**
2. **AWS Secret access key**
3. **AWS Region** (default `us-east-1`)
4. (Optional) **Hugging Face token** for gated models (e.g. Llama 3)

Magemaker will create/append to a `.env` file with the following variables:

```bash
AWS_ACCESS_KEY_ID="..."
AWS_SECRET_ACCESS_KEY="..."
SAGEMAKER_ROLE="arn:aws:iam::<account>:role/SageMakerRole"
AWS_DEFAULT_REGION="us-east-1"
HUGGING_FACE_HUB_KEY="..." # optional
```

<Warning>Never commit your `.env` file to version control!</Warning>

### 6 ‚Äì Verify Quotas
Even with the correct IAM role you might hit **instance-quota** limits when deploying large models.

1. Open the AWS Console ‚Üí **Service Quotas**.  
2. Filter for **Amazon SageMaker**.  
3. Check the *Instance* rows (e.g. `ml.g5.12xlarge`).  
4. Request an increase if necessary.
