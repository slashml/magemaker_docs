### Dockerized Usage

Magemaker supports running entirely within Docker, with the AWS CLI pre-installed and ready to use for streamlined deployments.

**Recommended Docker Setup:**

1. **.dockerignore**  
   Make sure to include a `.dockerignore` file that excludes local virtual environments, build artifacts, and configuration directories from your Docker build context:

   ```dockerignore
   venv/
   *.pyc
   __pycache__
   .git
   .env
   *.egg-info
   dist/
   build/
   ```

2. **Dockerfile**

   Magemaker provides a sample Dockerfile named `Dockerfile-server` that comes pre-configured with:
   
   - Python 3.11
   - All development dependencies installed through pip
   - AWS CLI (v2) pre-installed
   - Google Cloud SDK
   - Azure CLI
   - Bash entrypoint that automatically sets up environment variables such as AWS credentials (if mounted as volume)

   The relevant installation steps for AWS in the Dockerfile are summarized below:

   ```docker
   FROM python:3.11-slim

   # ...

   # Install AWS CLI
   RUN curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip" \
       && unzip awscliv2.zip \
       && ./aws/install \
       && rm awscliv2.zip \
       && rm -rf aws

   # Copy and setup entrypoint
   COPY magemaker/docker/entrypoint.sh /usr/local/bin/
   RUN chmod +x /usr/local/bin/entrypoint.sh

   ENTRYPOINT ["entrypoint.sh"]
   CMD ["bash"]
   ```

3. **Credential Handling**

   When running the Docker container, you can mount local AWS credentials for use _inside_ the container. The entrypoint script `/usr/local/bin/entrypoint.sh` will automatically configure environment variables if the credentials exist:

   ```sh
   #!/bin/bash
   set -e

   # Setup AWS credentials if mounted
   if [ -f "/root/.aws/credentials" ]; then
       export AWS_SHARED_CREDENTIALS_FILE="/root/.aws/credentials"
   fi

   # Setup GCP credentials if mounted
   if [ -f "/root/.config/gcloud/application_default_credentials.json" ]; then
       export GOOGLE_APPLICATION_CREDENTIALS="/root/.config/gcloud/application_default_credentials.json"
   fi

   exec "$@"
   ```

   > **Tip:** Pass your credentials into the container by mounting them at `/root/.aws/credentials`:
   >
   > ```sh
   > docker run -v ~/.aws/credentials:/root/.aws/credentials:ro magemaker
   > ```
   >
   > You can now use both the AWS CLI and magemaker commands from inside the container.

### AWS Account

<Steps>
<Step title="Create AWS Account">
Register for an [AWS account](https://aws.amazon.com/) and sign-in to the [console](https://console.aws.amazon.com/).
</Step>

<Step title="Navigate to IAM">
From the console, use the Search bar to find and select IAM (***do not use IAM Identity Center***, which is confusingly similar but a totally different system).

![Enter image alt description](../Images/muJ_Image_1.png)

You should see the following screen after clicking IAM.

![Enter image alt description](../Images/ldC_Image_2.png)
</Step>

<Step title="Create or Select User">
1. Select `Users` in the side panel
   
![Enter image alt description](../Images/QX4_Image_3.png)

2. Create a user if you don't already have one

![Enter image alt description](../Images/ly3_Image_4.png)
</Step>

<Step title="Add Required Permissions">
1. Click on "Add permissions"
   
![Enter image alt description](../Images/E7x_Image_5.png)

2. Select "Attach policies directly". Under permission policies, search for and tick the boxes for:
   - `AmazonSagemakerFullAccess`
   - `IAMFullAccess`
   - `ServiceQuotasFullAccess`

Then click Next.

![Enter image alt description](../Images/01X_Image_6.png)

The final list should look like the following:

![Enter image alt description](../Images/Dfp_Image_7.png)

Click "Create user" on the following screen.
</Step>

<Step title="Generate Access Keys">
1. Click the name of the user you've just created (or one that already exists)
2. Go to "Security Credentials" tab
3. Scroll down to "Access Keys" section
4. Click "Create access key"
5. Select Command Line Interface then click next

![Enter image alt description](../Images/BPP_Image_8.png)

Enter a description (this is optional, can leave blank). Then click next.

![Enter image alt description](../Images/gMD_Image_9.png)

**Store BOTH the Access Key and the Secret access key for the next step. Once you've saved both keys, click Done.**

![Enter image alt description](../Images/Gjw_Image_10.png)
</Step>
</Steps>
