### AWS CLI

<Note>To install the AWS CLI on macOS ≥ 13.6.6 running on Apple Silicon you *must* use a Rosetta terminal. Verify by running `arch` – it should print `i386` for a Rosetta session. Make sure you have the latest version of Xcode Command-Line Tools installed.</Note>

Follow the official guide to install the latest AWS CLI:

https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html

Once the CLI is installed and working, continue with the steps below.

---

### AWS Account

<Steps>
<Step title="Create AWS Account">
Register for an [AWS account](https://aws.amazon.com/) and sign-in to the [console](https://console.aws.amazon.com/).
</Step>

<Step title="Navigate to IAM">
From the console, use the search bar to find and select **IAM** (*do **not** use IAM Identity Center, which is different*).

![IAM Search](../Images/muJ_Image_1.png)

After clicking **IAM** you should see a screen similar to this:

![IAM Home](../Images/ldC_Image_2.png)
</Step>

<Step title="Create or Select User">
1. Select **Users** in the side-panel  
   ![Users](../Images/QX4_Image_3.png)

2. Create a user if you don't already have one  
   ![Create User](../Images/ly3_Image_4.png)
</Step>

<Step title="Add Required Permissions">
1. Click **Add permissions**  
   ![Add Permissions](../Images/E7x_Image_5.png)

2. Choose **Attach policies directly** and tick the following policies:
   - `AmazonSageMakerFullAccess`
   - `IAMFullAccess`
   - `ServiceQuotasFullAccess`

   Then click **Next**.

![Policies](../Images/01X_Image_6.png)

The selected policies should look like this:

![Policies List](../Images/Dfp_Image_7.png)

Click **Create user** to finish.
</Step>

<Step title="Generate Access Keys">
1. Click the name of the user you just created (or an existing user).  
2. Go to the **Security credentials** tab.  
3. Scroll to the **Access keys** section and click **Create access key**.  
4. Select **Command Line Interface** and click **Next**.  

![Create Access Key](../Images/BPP_Image_8.png)

Add an optional description, then **Next**.

![Key Description](../Images/gMD_Image_9.png)

**Save BOTH the Access Key ID and the Secret Access Key** – you will not be able to see the secret again after closing the dialog.

![Save Keys](../Images/Gjw_Image_10.png)
</Step>

<Step title="Create a SageMaker Execution Role (SAGEMAKER_ROLE)">
Magemaker requires an execution role so that SageMaker can access Amazon S3 and other services on your behalf. You have two options:

**Option A – Automatic (recommended)**
```bash
# Clone the repo or download the script first
bash magemaker/scripts/setup_role.sh
```
The script creates a role called `MagemakerExecutionRole` with the necessary trust relationship and policies (`AmazonSageMakerFullAccess`, `AmazonS3FullAccess`). Copy the generated **Role ARN** printed by the script.

**Option B – Manual**
1. In **IAM → Roles** click **Create role**.
2. Select **SageMaker** as the trusted entity type.
3. Attach the policies:  
   - `AmazonSageMakerFullAccess`  
   - `AmazonS3FullAccess`
4. Name the role (e.g. `MagemakerExecutionRole`) and create it.
5. Copy the role's **ARN** for the next step.
</Step>

<Step title="Populate .env">
Magemaker stores credentials in a local `.env` file. Create (or update) `.env` in your project root:

```bash
AWS_ACCESS_KEY_ID="<your-access-key-id>"
AWS_SECRET_ACCESS_KEY="<your-secret-access-key>"
AWS_DEFAULT_REGION="us-east-1"        # or your preferred region
SAGEMAKER_ROLE="arn:aws:iam::<acct>:role/MagemakerExecutionRole"

# Optional – required for gated Hugging Face models like Llama 3
HUGGING_FACE_HUB_KEY="<your-hf-token>"
```

<Warning>Never commit your `.env` file to version control!</Warning>
</Step>
</Steps>

## Next Step – Verify Configuration
Run the following command to verify everything works:

```bash
magemaker --cloud aws
```

The first run will test your credentials, create any missing resources, and confirm the role. After that, you are ready to deploy models.
