# AWS Configuration

To use AWS services through Magemaker—especially AWS SageMaker deployments—you must provide AWS credentials with sufficient permissions.

## Setting up AWS Credentials

Magemaker looks for standard AWS CLI credentials, either locally or in the container.

### 1. Creating an IAM User

1. Sign in to the AWS Console
2. Go to **IAM > Users**, and create a user with programmatic access
3. Attach the following permissions:
   - `AmazonSageMakerFullAccess`
   - `AmazonS3FullAccess`
   - Any other required policies

4. Download or note your `AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`.

### 2. Save Credentials (Locally)

Configure your local AWS CLI:

```bash
aws configure
```

This creates a credentials file at `~/.aws/credentials`.

### 3. Using Credentials in Docker

With Magemaker v0.3.0+, you can securely inject credentials into the container:

```bash
docker run -it \
  -v $HOME/.aws:/root/.aws:ro \
  magemaker-server
```

The container entrypoint will detect `/root/.aws/credentials` and set `AWS_SHARED_CREDENTIALS_FILE` automatically.

**Tip:** You may use a different profile by editing `~/.aws/credentials` or exporting `AWS_PROFILE`.

### 4. Testing Credentials

Use the `preflight.sh` script to validate connectivity:

```bash
bash magemaker/scripts/preflight.sh
```

It checks your permissions and prompts to install the AWS CLI if needed.

## Environment Variables

Magemaker will use:

- `AWS_SHARED_CREDENTIALS_FILE` if set
- Otherwise, the default credentials search path

Example (if not using the Docker entrypoint):

```bash
export AWS_SHARED_CREDENTIALS_FILE=/path/to/credentials
```

## Notes

- Make sure your user is in the correct AWS region (default: `us-west-2`; can be overridden by env var `AWS_REGION`)
- The deployment output will show where to find the configuration for querying the model:

    ```
    Important: Configuration saved at: /app/configs/<endpoint_name>.yml
    You'll need this path for querying the model later.
    ```

    Save the exported config path from your deployment logs!
