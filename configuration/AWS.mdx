## Overview

This guide walks you through setting up **AWS credentials, IAM roles, and quotas** so Magemaker can:

1. Create or reuse a **SageMaker execution role** (`SAGEMAKER_ROLE`).  
2. Spin-up training & inference instances.  
3. Access your S3 buckets for data & model artefacts.

<Note>
  All steps can be completed via the AWS Console **or** the CLI. Pick whichever you’re more comfortable with.
</Note>

---

### 1. Install & Configure the AWS CLI

Follow the official instructions:  
<https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html>

```bash
# After installation
aws configure                        # enter Access Key, Secret, default region
```

We recommend `us-east-1` because most JumpStart models are available there, but any region works as long as you have SageMaker quota.

---

### 2. Create an IAM User (Console)

<Steps>
  <Step title="Navigate to IAM">
    Open the console and search for **IAM** (not *IAM Identity Center*).
  </Step>
  <Step title="Add User & Permissions">
    1. Create a new user (or select an existing one).  
    2. Attach the following *managed policies*:
       - `AmazonSageMakerFullAccess`
       - `IAMFullAccess` (needed once to create the execution role)
       - `ServiceQuotasFullAccess` (for quota look-ups)
  </Step>
  <Step title="Generate Access Keys">
    Go to *Security Credentials* → *Access Keys* → **Create access key**.  
    Store **both** the *Access Key ID* and *Secret Access Key* – you’ll need them for `aws configure` *and* Magemaker.
  </Step>
</Steps>

<Warning>
  Do **not** commit your keys to GitHub.  Use a `.env` file (see Environment Variables).
</Warning>

---

### 3. Create / Verify a SageMaker Execution Role

Magemaker expects an **ARN** in `SAGEMAKER_ROLE` that has `AmazonSageMakerFullAccess`.

```bash
aws iam create-role \
  --role-name magemaker-sagemaker-role \
  --assume-role-policy-document file://trust-policy.json

aws iam attach-role-policy \
  --role-name magemaker-sagemaker-role \
  --policy-arn arn:aws:iam::aws:policy/AmazonSageMakerFullAccess
```

Copy the resulting role ARN into your `.env`.

---

### 4. Check / Request Service Quotas

1. Open **Service Quotas** in the console.  
2. Filter by *SageMaker*.  
3. Verify you have at least **2 × `ml.m5.xlarge` instances** (default).  
4. Request additional quota for GPU instances (e.g. `ml.g5.*`, `ml.p3.*`) if you plan to deploy large models.

<Note>
  Approval can take anywhere from minutes to 48 h depending on your AWS account history.
</Note>

---

## Final Checklist

- [x] AWS CLI installed & `aws configure` completed  
- [x] `.env` contains `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION_NAME`, `SAGEMAKER_ROLE`  
- [x] SageMaker quota for the instance types you plan to use

You’re now ready to run:

```bash
magemaker --cloud aws
```

This command validates your credentials, writes/updates the `.env`, and caches your default region.
