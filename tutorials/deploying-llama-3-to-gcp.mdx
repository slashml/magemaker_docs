## Introduction
This tutorial guides you through deploying Llama&nbsp;3 to Google Cloud Platform (GCP) Vertex AI using Magemaker and then querying the model. Make sure you have completed the [installation](installation) steps and configured GCP credentials with:

```bash
magemaker --cloud gcp
```

<Note>
You may need to request a quota increase for specific machine types and GPUs in the region where you plan to deploy the model. Check your GCP quotas before proceeding.
</Note>

## Step&nbsp;1: Setting Up Magemaker for GCP

Run the following command to configure Magemaker for Vertex AI deployment and create the required `.env` file (it will include `PROJECT_ID` and `GCLOUD_REGION`):

```bash
magemaker --cloud gcp
```

If you plan to deploy gated Hugging Face models such as Llama 3, add your HF token to the same `.env` file:

```bash
HUGGING_FACE_HUB_KEY="<your-hf-token>"
```

## Step&nbsp;2: YAML-based Deployment

For reproducible deployments, use a YAML configuration file:

```bash
magemaker --deploy .magemaker_config/llama3-gcp.yaml
```

Example YAML for GCP deployment:

```yaml
deployment: !Deployment
  destination: gcp
  endpoint_name: llama3-endpoint
  instance_type: g2-standard-12   # GPU-enabled machine type
  accelerator_type: NVIDIA_L4     # GPU type (L4)
  accelerator_count: 1            # Number of GPUs
  num_gpus: null                  # Optional override for multi-GPU setups
  quantization: null              # e.g. bitsandbytes | awq | null

models:
  - !Model
    id: meta-llama/Meta-Llama-3-8B-Instruct
    source: huggingface
    task: text-generation          # Optional but recommended
```

<Note>
For gated models like Llama 3, you must accept the model licence on Hugging Face and provide `HUGGING_FACE_HUB_KEY` in your `.env` file.
</Note>

### Selecting an Appropriate Instance
For Llama 3 (8 B), a `g2-standard-12` machine with an L4 GPU works well for most use-cases. Adjust the machine type, GPU, and `accelerator_count` based on your latency and cost requirements.

<Warning>
If you encounter quota issues, navigate to **IAM & Admin → Quotas** in the Google Cloud console and request an increase for the relevant GPU type in your chosen region.
</Warning>

## Step&nbsp;3: Querying the Deployed Model

After the deployment finishes, note the **endpoint ID** that Magemaker prints. You can query the model either via the interactive dropdown menu or programmatically.

### 3.a – Interactive Query
From the dropdown, select **Query a Model Endpoint**, choose your endpoint, type a prompt, and press **Enter** to receive the response.

![Query Endpoints](../Images/query-1.png)

### 3.b – Programmatic Query (REST)

Below is a minimal Python example that calls the Vertex AI endpoint using REST. It automatically retrieves credentials from your local gcloud setup or a service-account JSON file.

```python
from dotenv import dotenv_values
import google.auth
import google.auth.transport.requests
import requests


def query_vertexai_endpoint_rest(endpoint_id: str, prompt: str, token_path: str | None = None):
    """Query a Vertex AI endpoint created by Magemaker."""

    # Environment variables populated by `magemaker --cloud gcp`
    env = dotenv_values(".env")
    project_id = env.get("PROJECT_ID")
    location   = env.get("GCLOUD_REGION", "us-central1")

    # Load credentials
    if token_path:
        credentials, _ = google.auth.load_credentials_from_file(token_path)
    else:
        credentials, _ = google.auth.default()

    # Refresh access token
    credentials.refresh(google.auth.transport.requests.Request())

    headers = {
        "Authorization": f"Bearer {credentials.token}",
        "Content-Type": "application/json",
    }

    url = (
        f"https://{location}-aiplatform.googleapis.com/v1/projects/{project_id}/"
        f"locations/{location}/endpoints/{endpoint_id}:predict"
    )

    payload = {
        "instances": [
            {
                "inputs": prompt,
                "parameters": {
                    "max_new_tokens": 100,
                    "temperature": 0.7,
                    "top_p": 0.95,
                },
            }
        ]
    }

    response = requests.post(url, headers=headers, json=payload, timeout=60)
    response.raise_for_status()
    return response.json()


if __name__ == "__main__":
    ENDPOINT_ID = "your-endpoint-id-here"  # Replace with the real one
    prompt      = "What are you?"

    result = query_vertexai_endpoint_rest(ENDPOINT_ID, prompt)
    print(result)
```
