# Tutorial: Deploy Llama 3 to AWS SageMaker

This tutorial demonstrates deploying Llama 3 to SageMaker using Magemaker. Now updated to walk you through new CLI messages, config file output, and credential testing tools.

## Prerequisites
- AWS account with SageMaker and S3 permissions
- [AWS CLI configured](../configuration/AWS)
- Optional: Docker (with mounted credentials)
- Python 3.11+

## Step 1: Check Credentials & Install CLI

Run the preflight script:

```sh
bash magemaker/scripts/preflight.sh
```

This checks AWS CLI install and verifies credentials.

## Step 2: Deploy

```sh
magemaker deploy --provider aws --model-id meta-llama/Llama-3-8b-hf
```

## Step 3: Save the Deployment Config Path

After deployment, Magemaker prints:

```
Important: Configuration saved at: /app/configs/<endpoint_name>.yml
You'll need this path for querying the model later.
```

**Copy the file path!** You'll use it for queries.

## Step 4: Query the Model

```sh
magemaker query --endpoint <endpoint_name>
```

If no config is found, you will see a message showing the expected path. Make sure you have the `.yml` config in the directory indicated.

## Troubleshooting
- If credentials are wrong/missing, preflight will warn you
- If using Docker, mount `~/.aws` as described in [Installation](../installation)
- On query failure, check that the endpoint config exists at the required path
