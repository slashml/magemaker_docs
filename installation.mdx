<Note>
• macOS ≥ 13.6.6 is required.  
• Apple Silicon users must run the terminal in **Rosetta** (`arch` should print <code>i386</code>).
</Note>

## 1 – Install the Package
```bash
pip install magemaker
```

---

## 2 – Configure Cloud Providers
Use the built-in wizard—no manual JSON editing required:

```bash
# AWS
magemaker --cloud aws

# GCP
magemaker --cloud gcp

# Azure
magemaker --cloud azure

# All at once (creates a consolidated .env)
magemaker --cloud all
```

The wizard will:
1. Verify your CLI installation (<code>aws</code>, <code>gcloud</code>, or <code>az</code>)
2. Prompt for any missing credentials / regions
3. Write a **.env** with the keys shown below

---

## 3 – Environment Variables (.env)
Magemaker automatically loads a <code>.env</code> from your project root:

```bash
# AWS
AWS_ACCESS_KEY_ID="AKIA..."
AWS_SECRET_ACCESS_KEY="..."
SAGEMAKER_ROLE="arn:aws:iam::123456789012:role/MageMakerRole"
AWS_REGION_NAME="us-east-1"

# GCP
PROJECT_ID="your-project-id"
GCLOUD_REGION="us-central1"
# Optional – service-account key for CI/CD
GOOGLE_APPLICATION_CREDENTIALS="/path/to/key.json"

# Azure
AZURE_SUBSCRIPTION_ID="..."
AZURE_RESOURCE_GROUP="ml-resources"
AZURE_WORKSPACE_NAME="ml-workspace"
AZURE_REGION="eastus"

# Hugging Face (optional but required for gated models)
HUGGING_FACE_HUB_KEY="hf_..."
```

<Warning>Never commit <code>.env</code> to version control.</Warning>

---

## 4 – Verify Your Setup
```bash
magemaker --list   # should show endpoints or an empty list, not an error
```
