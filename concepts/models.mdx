## Supported Model Sources

Magemaker can deploy models from **multiple sources**.  The exact options depend on the cloud provider.

<CardGroup>
  <Card title="🤗 Hugging Face" icon="hugging-face" href="https://huggingface.co/models">
    All public Hugging Face models are supported on AWS, GCP & Azure.
  </Card>
  <Card title="AWS JumpStart" icon="aws">
    Browse and deploy SageMaker JumpStart models (classification, text-gen, vision, etc.) **directly from the CLI**.
  </Card>
  <Card title="Custom (S3 / Local)" icon="folder">
    Point Magemaker at a local directory or an S3 URI – handy for fine-tuned checkpoints or proprietary models (AWS only for now).
  </Card>
</CardGroup>

<Note>
  JumpStart & custom S3 models are currently **AWS-only**.  GCP Model Garden / Azure ML Registry integration is in progress.
</Note>

## Instance Recommendations

Below are **rule-of-thumb** suggestions.  Always check actual GPU/CPU requirements of your model.

### AWS SageMaker

| Model Size        | Instance                | GPUs | Notes |
|-------------------|-------------------------|------|-------|
| ≤ 1 B parameters  | `ml.m5.xlarge`          | –    | CPU-only, cheapest |
| 1–7 B parameters  | `ml.g4dn.xlarge`        | 1× T4 | good balance |
| 7–70 B parameters | `ml.g5.12xlarge`        | 4× A10G | HF Llama-3-8B works here |

### GCP Vertex AI

| Model Size | Machine Type       | GPU            |
|------------|--------------------|----------------|
| small      | `n1-standard-4`    | –              |
| medium     | `n1-standard-8`    | 1× T4          |
| large      | `a2-highgpu-1g`    | 1× A100        |

### Azure ML

| Model Size | VM Size                | GPU        |
|------------|------------------------|------------|
| small      | `Standard_DS3_v2`      | –          |
| medium     | `Standard_NC6s_v3`     | 1× V100    |
| large      | `Standard_NC24ads_A100_v4` | 4× A100 |

<Warning>
  GPU names differ across clouds (A10G ≈ L4 ≈ V100). Always double check memory & cost.
</Warning>

## Example Deployments

```yaml
# Hugging Face model → AWS
models:
- !Model
  id: facebook/opt-125m
  source: huggingface

deployment: !Deployment
  destination: aws
  instance_type: ml.g4dn.xlarge
```

```yaml
# JumpStart model → AWS (interactive CLI can search for IDs)
models:
- !Model
  id: jumpstart-lang-textclassification-bert-base
  source: sagemaker

deployment: !Deployment
  destination: aws
```

```yaml
# Hugging Face model → GCP
models:
- !Model
  id: meta-llama/Meta-Llama-3-8B-Instruct
  source: huggingface

deployment: !Deployment
  destination: gcp
  machine_type: a2-highgpu-1g
  accelerator_type: NVIDIA_A100
  accelerator_count: 1
```

<Note>
  Azure uses *different* model IDs for Hugging Face.  Use the Azure ML **Model Catalog** to copy the correct ID.
</Note>

## Best Practices

1. **Start small** – prototype with CPU instances, then scale up.  
2. **Monitor costs & quotas** – big GPUs can be expensive and scarce.  
3. **Version your YAML** – keep configs in Git for reproducibility.

## Troubleshooting

| Issue | Possible Cause | Fix |
|-------|----------------|-----|
| `RuntimeError: CUDA out of memory` | Instance has insufficient GPU RAM | Pick a bigger instance / enable quantization |
| Deployment stuck in *Creating* | Region capacity, missing quota | Try another region or request quota increase |
| 403 accessing model | Gated HF model | Accept terms on HF & set `HUGGING_FACE_HUB_KEY` in `.env` |
