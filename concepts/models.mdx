## Supported Model Sources

Magemaker can deploy models from more than one source. Specify the source in the `!Model` block with the `source` field.

<CardGroup>
  <Card title="Hugging Face" icon="pen-to-square" href="https://huggingface.co/models">
    - Text-generation, feature-extraction, etc.<br/>
    - Works across **AWS**, **GCP**, and **Azure** (IDs differ on Azure!)
  </Card>
  <Card title="AWS JumpStart" icon="aws" href="https://docs.aws.amazon.com/sagemaker/latest/dg/jumpstart.html">
    - Foundation models curated by AWS<br/>
    - Deployable **only on SageMaker** for now<br/>
    - Includes proprietary & open-source models such as Falcon-40B-Instruct, Mistral-7B, etc.
  </Card>
</CardGroup>

<Note>
Add more sources by extending the `source` field (e.g. `sagemaker`, `huggingface`). Support for GCP Model Garden and Azure OpenAI is on the roadmap.
</Note>

### Example `!Model` blocks

```yaml
# Hugging Face model
- !Model
  id: meta-llama/Meta-Llama-3-8B-Instruct
  source: huggingface
```

```yaml
# AWS JumpStart model (ID is the JumpStart model ID)
- !Model
  id: huggingface-llama-2-7b-f
  source: sagemaker   # JumpStart models use the "sagemaker" source
```

## Instance-Type Recommendations (by Cloud)

#### AWS SageMaker
1. **Small** – `ml.m5.xlarge`
2. **Medium** – `ml.g4dn.xlarge`
3. **Large** – `ml.g5.12xlarge` (4× A10G GPU)

#### GCP Vertex AI
1. **Small** – `n1-standard-4`
2. **Medium** – `n1-standard-8` + `NVIDIA_TESLA_T4`
3. **Large** – `a2-highgpu-1g`

#### Azure ML
1. **Small** – `Standard_DS3_v2`
2. **Medium** – `Standard_NC6s_v3`
3. **Large** – `Standard_ND40rs_v2`

## Example Deployments

```yaml
# SageMaker – Hugging Face model
models:
- !Model
  id: facebook/opt-125m
  source: huggingface

deployment: !Deployment
  destination: aws
```

```yaml
# SageMaker – JumpStart model
models:
- !Model
  id: huggingface-llama-2-7b-f   # JumpStart model ID
  source: sagemaker

deployment: !Deployment
  destination: aws
```

```yaml
# Vertex AI – Hugging Face model
models:
- !Model
  id: facebook/opt-125m
  source: huggingface

deployment: !Deployment
  destination: gcp
```

```yaml
# Azure ML – Hugging Face model (note different ID format)
models:
- !Model
  id: facebook-opt-125m
  source: huggingface

deployment: !Deployment
  destination: azure
```

<Note>
Azure uses its own model-catalog IDs. Follow the steps in the **Quick-Start** guide to locate the correct ID.
</Note>

## Model Configuration Reference

### Basic

```yaml
- !Model
  id: <model-id>
  source: huggingface | sagemaker
```

### Advanced

```yaml
- !Model
  id: <model-id>
  source: huggingface
  predict:            # Optional runtime parameters
    temperature: 0.7
    top_p: 0.9
    top_k: 50
    max_new_tokens: 500
```

## Best Practices

1. **Model Selection** – Evaluate cost & latency per cloud.
2. **Cost Management** – Tear down unused endpoints and set up budget alerts.
3. **Performance Tuning** – Adjust instance/GPU sizes; monitor logs & metrics.

---

## Troubleshooting

1. **Quota Errors** – Request increases for GPU/instance quotas.
2. **Auth Errors** – Verify cloud credentials & HF tokens.
3. **Slow Start-up** – Large models may take >15 min; watch endpoint logs for progress.
