## About Magemaker

Magemaker is a Python tool that simplifies the process of deploying open source AI models to your preferred cloud provider. Instead of spending hours digging through documentation to figure out how to get cloud deployments working, Magemaker lets you deploy open source AI models directly from the command line.

Choose a model from Hugging Face and Magemaker will spin up the appropriate cloud instance with a ready-to-query endpoint in minutes. Currently supported cloud platforms:
- AWS SageMaker
- Google Cloud Vertex AI
- Azure Machine Learning

## Getting Started

### Prerequisites

* Python
* Cloud provider account (AWS/GCP/Azure)
* Cloud provider quota for compute instances
* Certain Hugging Face models (e.g. Llama2) require an access token ([hf docs](https://huggingface.co/docs/hub/en/models-gated#access-gated-models-as-a-user))

### Installation

Install via pip:

```sh
pip install magemaker
```

Run Magemaker:

```sh
magemaker
```

<Note>
On first run, Magemaker will guide you through configuring your chosen cloud provider. You'll be prompted to enter the necessary credentials.
</Note>

## Using Magemaker

### Interactive Deployment

Run `magemaker` to access an interactive menu where you can:
- Choose your cloud provider
- Select from available models
- Configure deployment settings
- Monitor deployment progress

### YAML-based Deployment

For reproducible deployments, use YAML configuration:

```sh
magemaker --deploy .magemaker_config/your-model.yaml
```

Example YAML for AWS deployment:

```yaml
deployment: !Deployment
  destination: aws
  endpoint_name: test-bert-uncased
  instance_count: 1
  instance_type: ml.m5.xlarge

models:
- !Model
  id: google-bert/bert-base-uncased
  source: huggingface
```

For GCP Vertex AI:

```yaml
deployment: !Deployment
  destination: gcp
  endpoint_name: test-bert-gcp
  instance_count: 1
  machine_type: n1-standard-4

models:
- !Model
  id: google-bert/bert-base-uncased
  source: huggingface
```

For Azure ML:

```yaml
deployment: !Deployment
  destination: azure
  endpoint_name: test-bert-azure
  instance_count: 1
  instance_type: Standard_DS3_v2

models:
- !Model
  id: google-bert/bert-base-uncased
  source: huggingface
```

### Model Fine-tuning

Fine-tune models using the `train` command:

```sh
magemaker --train .magemaker_config/train-config.yaml
```

Example training configuration:

```yaml
training: !Training
  destination: aws  # or gcp, azure
  instance_type: ml.p3.2xlarge  # varies by cloud provider
  instance_count: 1
  training_input_path: s3://your-bucket/data.csv
  hyperparameters: !Hyperparameters
    epochs: 3
    per_device_train_batch_size: 32
    learning_rate: 2e-5
```

## Known issues

- Querying within Magemaker currently only works with text-based models
- Deleting a model is not instant, it may show up briefly after deletion
- Deploying the same model within the same minute will break
