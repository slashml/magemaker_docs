# MageMaker Command-Line Interface Extension: YAML-Based Querying

## Overview

This update introduces a new command-line argument, `--query`, to the MageMaker command-line interface. The `--query` argument allows users to specify a path to a YAML query configuration file. This enhancement provides a streamlined, user-friendly method to send inference requests to deployed models using configurable YAML files.

---

## Detailed Description of Changes

### Addition of the `--query` Argument

- **New Argument:** `--query`
- **Purpose:** Provides a way to query deployed models using configuration stored in a YAML file.
- **Type:** String (expects a file path)
- **Help Message:** `"path to YAML query configuration file"`
- **Action:** Uses a custom action handler called `'query'` (implementation not shown in this diff).

#### Argument Details

| Argument      | Type   | Required | Description                                       |
|---------------|--------|----------|---------------------------------------------------|
| `--query`     | str    | No       | Path to the YAML file with query configuration.    |

With this addition, users can perform batch or complex queries to their deployed models in a standardized, maintainable way.

---

## Usage Examples

### YAML-Based Querying (Recommended)

MageMaker supports querying deployed models using YAML configuration files. This provides a convenient and reproducible way to send inference requests to your endpoints, especially for batch or structured inputs.

#### Command Structure

```bash
magemaker --query .magemaker_config/your-model.yaml
```

- Replace `.magemaker_config/your-model.yaml` with the path to your actual YAML query configuration file.

#### Example

Suppose you have a YAML file, `.magemaker_config/gpt-inference.yaml`, defining the input payload for a query:

```yaml
endpoint: gpt-endpoint
inputs:
  - prompt: "Explain the theory of relativity."
    max_tokens: 150
```

Execute the following command in your terminal:

```bash
magemaker --query .magemaker_config/gpt-inference.yaml
```

MageMaker will read the configuration and send the query to the specified deployed model endpoint.

---

## Summary

The addition of the `--query` argument makes it easier to interact with and test deployed MageMaker endpoints. This YAML-based approach:
- Standardizes query configuration,
- Improves reproducibility,
- Is suitable for scripting batch requests,
- Integrates well with infrastructure-as-code and CI/CD practices.

If you have existing YAML query files, you can use this feature immediately by providing the file path; otherwise, see MageMaker's documentation for guidance on creating compatible YAML query files.

### YAML-based Deployment (Recommended)

For reproducible deployments, use YAML configuration:

```sh
magemaker --deploy .magemaker_config/your-model.yaml
```

Example YAML for AWS deployment:

```yaml
deployment: !Deployment
  destination: aws 
  endpoint_name: facebook-opt-test
  instance_count: 1
  instance_type: ml.m5.xlarge
  num_gpus: null
  quantization: null
models:
  - !Model
    id: facebook/opt-125m
    location: null
    predict: null
    source: huggingface
    task: text-generation
    version: null
```

For GCP Vertex AI:

```yaml
deployment: !Deployment
  destination: gcp
  endpoint_name: facebook-opt-test
  accelerator_count: 1
  instance_type: g2-standard-12
  accelerator_type: NVIDIA_L4
  num_gpus: null
  quantization: null

models:
  - !Model
    id: facebook/opt-125m
    location: null
    predict: null
    source: huggingface
    task: null
    version: null
```

For Azure ML:

```yaml
deployment: !Deployment
  destination: azure
  endpoint_name: facebook-opt-test
  instance_count: 1
  instance_type: Standard_DS3_v2
models:
  - !Model
    id: facebook--opt-125m
    location: null
    predict: null
    source: huggingface
    task: text-generation
    version: null
```
<Note>
  The model ids for Azure are different from AWS and GCP. Make sure to use the one provided by Azure in the Azure Model Catalog. 

  To find the relevant model id, follow the following steps

      <Step title="Select Hugging Face in the Collections List">

    	![Azure ML Creation](../Images/hugging-face.png)
    </Step>

  </Steps>
</Note>



### Model Fine-tuning

Fine-tune models using the `train` command:

```sh
magemaker --train .magemaker_config/train-config.yaml
```

Example training configuration:

```yaml
training: !Training
  destination: aws # or gcp, azure
  instance_type: ml.p3.2xlarge # varies by cloud provider
  instance_count: 1
  training_input_path: s3://your-bucket/data.csv
  hyperparameters: !Hyperparameters
    epochs: 3
    per_device_train_batch_size: 32
    learning_rate: 2e-5
```
