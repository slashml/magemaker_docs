## Interactive View

1. Run Magemaker with your desired cloud provider:

```sh
magemaker --cloud [aws|gcp|azure|all]
```

Supported providers:

- `--cloud aws` AWS SageMaker deployment
- `--cloud gcp` Google Cloud Vertex AI deployment
- `--cloud azure` Azure Machine Learning deployment
- `--cloud all` Configure all three providers at the same time

## MageMaker Query Feature via YAML Configuration

MageMaker has introduced support for querying deployed models using a YAML-based configuration file. This feature streamlines interaction with endpoints and enhances flexibility by allowing users to specify query parameters in a structured, human-readable format.

---

### High-Level Overview

The latest update adds a new command-line argument, `--query`, to the MageMaker CLI. This argument allows users to provide the path to a YAML configuration file containing query specifications. The primary goal is to enable users to send inference requests and interact with deployed models using straightforward YAML configuration files.

---

### Detailed Change Description

#### 1. New Command-Line Argument: `--query`

- **Option Name**: `--query`
- **Purpose**: Allows users to specify a YAML configuration file that defines a query for a deployed model.
- **Argument Type**: `str` (expects a string path to the YAML file)
- **Help Text**: Describes that the argument points to a YAML query configuration file.
- **Integration**: Added to the argument parser, alongside other deployment and management flags.

##### Example Addition in Code
```python
parser.add_argument(
    '--query',
    action='query',
    help="path to YAML query configuration file",
    type=str
)
```
> **Note**: The `action='query'` indicates a custom action, which should be implemented elsewhere in the codebase.


---

### YAML Query Configuration

Your YAML file should specify parameters relevant to the model you want to query, such as endpoint name, input data, and other optional settings. This promotes clear, version-controlled, and reusable query specifications.

#### Example YAML (your-model.yaml)
```yaml
endpoint: my-sentiment-endpoint
inputs:
  text: "I love using MageMaker!"
options:
  content_type: application/json
```

---

### Usage Examples

You can use the `--query` argument from the command line as follows:

```bash
magemaker --query path/to/your-model.yaml
```
or, for typical project setups:
```bash
magemaker --query .magemaker_config/your-model.yaml
```

When this command is run, MageMaker reads the YAML configuration file and conducts the inference or query as described within the file.

---

### Summary Table

| Argument   | Value           | Description                                   |
|------------|-----------------|-----------------------------------------------|
| `--query`  | `<YAML_PATH>`   | Path to YAML query configuration file         |

---

### Best Practices

- **Keep configuration files versioned** within your repository to promote reproducibility.
- **Validate** YAML files with your team or CI pipelines to avoid formatting or logical errors.
- **Document expected structure** of the YAML file for your models, so team members can craft queries easily.

---

With this new YAML-based querying capability, MageMaker empowers you to script or automate inference and other model interactions in a robust, portable, and user-friendly manner. 




### YAML-based Deployment (Recommended)

For reproducible deployments, use YAML configuration:

```sh
magemaker --deploy .magemaker_config/your-model.yaml
```

Example YAML for AWS deployment:

```yaml
deployment: !Deployment
  destination: aws 
  endpoint_name: facebook-opt-test
  instance_count: 1
  instance_type: ml.m5.xlarge
  num_gpus: null
  quantization: null
models:
  - !Model
    id: facebook/opt-125m
    location: null
    predict: null
    source: huggingface
    task: text-generation
    version: null
```

For GCP Vertex AI:

```yaml
deployment: !Deployment
  destination: gcp
  endpoint_name: facebook-opt-test
  accelerator_count: 1
  instance_type: g2-standard-12
  accelerator_type: NVIDIA_L4
  num_gpus: null
  quantization: null

models:
  - !Model
    id: facebook/opt-125m
    location: null
    predict: null
    source: huggingface
    task: null
    version: null
```

For Azure ML:

```yaml
deployment: !Deployment
  destination: azure
  endpoint_name: facebook-opt-test
  instance_count: 1
  instance_type: Standard_DS3_v2
models:
  - !Model
    id: facebook--opt-125m
    location: null
    predict: null
    source: huggingface
    task: text-generation
    version: null
```
<Note>
  The model ids for Azure are different from AWS and GCP. Make sure to use the one provided by Azure in the Azure Model Catalog. 

  To find the relevant model id, follow the following steps
  <Steps>
    <Step title="Go to your workspace studio">
      Find the workpsace in the Azure portal and click on the studio url provided. Click on the `Model Catalog` on the left side bar
    	![Azure ML Creation](../Images/workspace-studio.png)
    </Step>

      <Step title="Select Hugging Face in the Collections List">
      Select Hugging-Face from the collections list. The id of the model card is the id you need to use in the yaml file
    	![Azure ML Creation](../Images/hugging-face.png)
    </Step>

  </Steps>
</Note>



### Model Fine-tuning

Fine-tune models using the `train` command:

```sh
magemaker --train .magemaker_config/train-config.yaml
```

Example training configuration:

```yaml
training: !Training
  destination: aws # or gcp, azure
  instance_type: ml.p3.2xlarge # varies by cloud provider
  instance_count: 1
  training_input_path: s3://your-bucket/data.csv
  hyperparameters: !Hyperparameters
    epochs: 3
    per_device_train_batch_size: 32
    learning_rate: 2e-5
```
