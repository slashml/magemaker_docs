## About Magemaker

Magemaker is a Python tool that simplifies the process of deploying open source AI models to your preferred cloud provider. Instead of spending hours digging through documentation, Magemaker lets you deploy Hugging Face models directly to AWS SageMaker, Google Cloud Vertex AI, or Azure Machine Learning.

## Prerequisites

- Python 3.11+
- Cloud provider account(s):
  - AWS for SageMaker
  - GCP for Vertex AI
  - Azure for Azure ML
- Cloud CLI tools:
  - AWS CLI (optional)
  - Google Cloud SDK for GCP
  - Azure CLI for Azure
- Hugging Face account (for access to models)

## Installation

Install via pip:

```sh
pip install magemaker
```

## First-time Setup

<Note>
  The first time you run Magemaker, it will need to perform initial
  configuration which may take 1-2 minutes. This includes setting up cloud
  provider credentials and creating necessary configuration files. You'll be
  prompted to enter the necessary credentials.

  You can follow the steps in the following doc to properly set up your initial configuration [Google Doc](https://docs.google.com/document/d/1NvA6uZmppsYzaOdkcgNTRl7Nb4LbpP9Koc4H_t5xNSg/edit?tab=t.0#heading=h.farbxuv3zrzm)
</Note>

1. Run Magemaker with your desired cloud provider:

```sh
magemaker --cloud [aws|gcp|azure|all]
```

2. Magemaker will:
   - Configure the specified cloud provider
   - Set up necessary credentials
   - Prepare environment for deployment

The `--cloud` flag allows direct configuration of your preferred cloud provider, streamlining the setup process.

## Using Magemaker

### Interactive Deployment

Run Magemaker with a cloud provider flag `--cloud` to access the deployment menu:

```sh
magemaker --cloud gcp
```

This approach allows you to:

- Directly target a specific cloud provider
- Quickly configure and deploy models for GCP, AWS, or Azure

Supported providers:

- `--cloud aws` AWS SageMaker deployment
- `--cloud gcp` Google Cloud Vertex AI deployment
- `--cloud azure` Azure Machine Learning deployment
- `--cloud all` Configure all three providers at the same time

### YAML-based Deployment

For reproducible deployments, use YAML configuration:

```sh
magemaker --deploy .magemaker_config/your-model.yaml
```

Example YAML for AWS deployment:

```yaml
deployment: !Deployment
  destination: aws 
  endpoint_name: facebook--opt-125m-202410251736
  instance_count: 1
  instance_type: ml.m5.xlarge
  num_gpus: null
  quantization: null
models:
  - !Model
    id: facebook/opt-125m
    location: null
    predict: null
    source: huggingface
    task: text-generation
    version: null
```

For GCP Vertex AI:

```yaml
deployment: !Deployment
  destination: gcp
  endpoint_name: test-endpoint-12
  accelerator_count: 1
  instance_type: g2-standard-12
  accelerator_type: NVIDIA_L4
  num_gpus: null
  quantization: null

models:
  - !Model
    id: facebook/opt-125m
    location: null
    predict: null
    source: huggingface
    task: null
    version: null
```

For Azure ML:

```yaml
deployment: !Deployment
  destination: azure
  endpoint_name: facebook--opt-125m-202410251736
  instance_count: 1
  instance_type: Standard_DS3_v2
models:
  - !Model
    id: facebook-opt-125m
    location: null
    predict: null
    source: huggingface
    task: text-generation
    version: null
```

### Model Fine-tuning

Fine-tune models using the `train` command:

```sh
magemaker --train .magemaker_config/train-config.yaml
```

Example training configuration:

```yaml
training: !Training
  destination: aws # or gcp, azure
  instance_type: ml.p3.2xlarge # varies by cloud provider
  instance_count: 1
  training_input_path: s3://your-bucket/data.csv
  hyperparameters: !Hyperparameters
    epochs: 3
    per_device_train_batch_size: 32
    learning_rate: 2e-5
```

## Known issues

- Querying within Magemaker currently only works with text-based models
- Deleting a model is not instant, it may show up briefly after deletion
- Deploying the same model within the same minute will break
