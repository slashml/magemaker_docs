# Quick Start

Get up and running with Magemaker for cloud LLM deployment in just a few steps! Now with automated credential testing, Dockerfile improvements, and clearer config guidance.

## 1. Install Magemaker

```bash
pip install magemaker
```

## 2. Set up Your Cloud Credentials

- **AWS**: Set up with `aws configure` (or mount `~/.aws` folder in Docker)
- **GCP**: Authenticate with `gcloud auth application-default login` (or mount `$HOME/.config/gcloud`)
- **Azure**: See [Azure config docs](configuration/Azure)

## 3. Check Your Setup (Recommended)

Run the pre-flight script to test credentials and ensure required cloud CLIs are installed:

```bash
bash magemaker/scripts/preflight.sh
```

Follow the prompts. The script checks and (if needed) installs required CLI tools and verifies cloud access.

## 4. Deploy a Model

Deploy a HuggingFace model to AWS SageMaker:

```bash
magemaker deploy --provider aws --model-id google-bert/bert-base-uncased
```

The CLI will print logs, including (new!):

```bash
Important: Configuration saved at: /your/path/configs/<endpoint_name>.yml
You'll need this path for querying the model later.
```

**Note:** Save/copy the config file path shown after deployment.

## 5. Query the Model

You can now query your deployed model by referring to the config file. For example:

```bash
magemaker query --endpoint <endpoint_name>
```

If the config file for your endpoint is missing, Magemaker will now print a detailed error message with the expected file location.
